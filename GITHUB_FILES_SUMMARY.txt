1. Repository Size: Keep it lightweight and fast to clone
   - PDFs/DOCX/PPTX are large binary files (>7 MB total)
   - GitHub works best with code, not large binaries

2. Regenerable Content: Anyone can recreate them
   - Run: python run_full_evaluation.py
   - Takes ~2 minutes to regenerate all reports

3. Version Control: Binary files don't diff well
   - Git can't show meaningful diffs for PDF/DOCX
   - Code changes are clear, binary changes are opaque

4. Professional Practice: Industry standard
   - Real projects separate code from generated outputs
   - CI/CD pipelines regenerate reports on demand
